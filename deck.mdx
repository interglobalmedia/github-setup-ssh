<ul style={{fontSize: '2.5rem', marginTop: '-0.125rem'}}>
    <li>Credentials are not necessary when using <code>SSH keys</code>. SSH consists of a <code>public</code> and <code>private key</code>. The <code>public key</code> is <i>stored</i> on <code>Github</code>, and the <code>private key</code> is <i>stored</i> in a safe place on your <code>computer</code>.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Why SSH instead of HTTPS?</h3>

<ul style={{fontSize: '2.5rem'}}>

</ul>

<ul style={{fontSize: '2.5rem'}}>
    <li>When paired properly, Github recognizes that your computer is the only one that can push to the repositories associated with your Github profile.</li>
</ul>

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Why SSH instead of HTTPS?</h3>

<ul style={{fontSize: '2.5rem'}}>

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Why SSH instead of HTTPS?</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>Getting to understand how SSH Technology works in the process of setting up a profile on Github makes the process of pushing projects to Github easier.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>What is SSH?</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>SSH stands for <code>Secure Shell</code>, and it is a method for secure remote login from one computer to another.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Why Use SSH?</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>It provides several alternative options for strong authentication, and it protects the communication's security and integrity with strong encryption.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Why Use SSH?</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>SSH is a secure alternative to the non-protected login protocals such as <a href="https://www.ssh.com/ssh/telnet">telnet</a> or <a href="https://www.ssh.com/ssh/rlogin">rlogin</a> and insecure file transfer methods such as <a href="https://www.ssh.com/ssh/ftp/">FTP</a>.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Authentication with SSH</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>Authentication works through the exchange of public and private keys. These keys are stored in files.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Authentication with SSH</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>A public key is stored on a trusted server such as Github, and a private key is stored on your device.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Using SSH</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>In order be able to use SSH, you have to generate a public and private key pair.</li>
    <li>The pair is usually stored in an ~/.ssh directory which only exists if you already have created SSH keys in the past.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Using SSH</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>Before creating your key pair, first check if you already have a key pair by cding into ~/.ssh. If you do have a key pair, skip to the last step.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Using SSH</h3>

<div style={{fontSize: '2.5rem', background: '#42ff71', padding: '5px 10px'}}>
    <code style={{background: 'transparent'}}>
        cd ~/.ssh<br/>
        ls<br/>
        # if you get anything back that ends in `.pub`, you might have SSH keys!<br/>
    </code>
</div>

---

<h3 style={{textAlign: 'center', fontSize: '3rem', marginTop: '-2rem'}}>Creating Your Keys</h3>

<p style={{fontSize: '1.75rem'}}>Use the following commands to generate a new key pair:</p>

<div style={{background: '#42ff71', padding: '0px 10px 5px'}}>
    <code style={{fontSize: '1.5rem'}}>
        ssh-keygen -t rsa -C "youremail@example.com"<br/>
        # Enter the file in which to save the key pair(/Users/username/.ssh/id_rsa): [Press Enter]<br/>
        # Enter passphrase (empty for no passphrase): [Type a passphrase]<br/>
        # Enter the same passphrase again: [Type passphrase]
    </code>
</div>

---

<h3 style={{textAlign: 'center', fontSize: '3.5rem', marginTop: '-2rem'}}>Creating Your Keys</h3>

<p style={{fontSize: '2.5rem'}}>If everything goes well, you'll receive a message stating that your keys have been saved.</p>

---

<h3 style={{textAlign: 'center', fontSize: '3rem', marginTop: '-2rem'}}>Adding Your New Key to ssh-agent</h3>

<ul style={{fontSize: '2rem'}}>
    <li>
        <code>ssh-agent</code> is a helper program that keeps track of a user's <code>identity keys</code> and <code>passphrases</code>.
    </li>
    <li>The agent can then use the keys to log into other servers without having the user type in a password or passphrase again. This implements a form of single sign-on (SSO).</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3rem', marginTop: '-2rem'}}>Adding Your New Key to ssh-agent</h3>

<ul style={{fontSize: '2.5rem'}}>
    <li>The SSH agent is used for SSH public key authentication. It uses SSH keys for authentication. Users can create SSH keys using the ssh-keygen command and install them on servers using the ssh-copy-id command.</li>
</ul>

---

<h3 style={{textAlign: 'center', fontSize: '3rem', marginTop: '-2rem'}}>Using ssh-agent</h3>

<div style={{fontSize: '2.5rem', background: '#42ff71', padding: '0px 10px 5px'}}>
    <code style={{background: 'transparent', fontSize: '1.5rem'}}>
    # start the ssh-agent in the background<br/>
    $ eval "$(ssh-agent -s)"<br/>
    # Agent pid 34234<br/>
    # Add your key<br/>
    $ ssh-add ~/.ssh/id_rsa<br/>
    </code>
</div>

---

<h3 style={{textAlign: 'center', fontSize: '3rem', marginTop: '-2rem'}}>Final Steps: Copying Your Key</h3>

<div style={{fontSize: '2.5rem', background: 'transparent', padding: '0px 10px 5px'}}>
    <p style={{fontSize: '2rem'}}>
        Now that you have generated and stored your SSH key pair, you need to copy your public key. The instructions assume that you have named your public key <code>id_rsa.pub</code> (the default name).
    </p>
    <span style={{fontSize: '2rem', background: '#42ff71', color: '#000'}}>Please ignore the single quotes!</span>

'pbcopy < ~/.ssh/id_rsa.pub'
    
</div>

---

<h3 style={{textAlign: 'center', fontSize: '2.75rem', marginTop: '-2rem'}}>Final Steps: Adding Keys to Your Account(s)</h3>

<ul style={{fontSize: '2.25rem'}}>
    <li>If you are adding the copied public key to Github, go to Settings -> SSH Keys -> Add SSH Key on Github.com.</li>
    <li>Paste what you have copied into the text area, and hit the "Add Key" button.</li>
    <li>Now you can start using the SSH URL on Github instead of HTTPS.</li>
</ul>
